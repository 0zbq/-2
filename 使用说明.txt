1.git clone + 地址   或者    先创建一个文件夹，然后git init
2.git add .                                将当前路径下所有文件和非空的文件夹设置为准备提交的状态
3.git commit -m "备注"         将准备提交的内容提交到仓库中
说明：git仓库内存放了这个项目所有原代码，
           所谓git仓库就是指项目文件夹里的.git文件夹
           尽管知道.git文件夹就是我们要操作的对象，但是我们不能直接对.git文件夹修改
           提交成功后git会把源代码以数据库的形式保存在仓库中

4.查看
     git log                                                  查看提交的历史记录
     git log --stat                                       查看提交的历史记录并查看每次提交所修改的文件
     git diff 某次提交的commit_ID           查看ID所对应的提交所修改的内容
     git branch                                           查看当前项目有哪些分支     

5.切换
     git checkout   分支名称                     从当前分支切换到指定的分支 
     git checkout   -                                   快速切换回前一个分支
     git checkout -b 新的分支名               创建一个新分支并立即切换到新创建的分支，实现多版本功能
     git checkout   commit_ID                 切换到特定的提交状态,这将使你进入"分离头指针"状态，只能查看历史记录，不建议在分离头指针状态下工作，因为更改可能会丢失。


6.恢复
     git checkout HEAD 要恢复的文件         从最后一次的提交将要恢复的文件复制到工作区
     git checkout -- 要恢复的文件                 将工作区指定文件恢复到最近一次提交时的状态丢弃所有未提交的更改


7.首先切换至分支的上一分支，然后git merge 分支名            将分支中的内容合并到（添加到）上一分支，防止多人操作的混乱，个人操作不需要此步
           

8.链接github
      git remote+自定义别名+地址             使本地仓库与GitHub上的远程仓库建立联系
      git push+别名+要上传的分支               上传本地仓库中的分支
      git pull+别名+要拉取得分支                  拉取远程仓库中的分支
